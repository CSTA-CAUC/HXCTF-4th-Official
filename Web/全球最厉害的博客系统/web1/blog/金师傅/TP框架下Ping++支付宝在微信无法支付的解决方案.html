TP框架下Ping++支付宝在微信无法支付的解决方案
  • Kylin King
  • 二月 19 2016
  • 前端
背景

众所周知的，A厂和T厂就支付方面是竞争对手。双方互相封杀对方的支付产品，苦逼了我们Coder啊！~

问题描述

Wap版用在微信公众号上，但支付接的是ping++的支付宝，在微信上一支付，就出现这个问题:

这就造成了超级不好的用户体验，复制再粘贴的操作不知对于一个本想买我们课程的家长来说是多么繁琐。

解决方案

Google了一下，有不少大神用的是iframe对解决的，但项目接的是ping++，不能直接拿到url，且项目也比较紧急(明天开始推C端)，这个方案暂时先不考虑。所以就用了另一个方案：

在支付宝帮助中心，提供了一个简易的demo(真的是太简易了，且我用的ping++，很多不一样的地方)。

看了一下，大概说的意思是：ap.js文件用于捕获被微信屏蔽的支付宝地址，pay.html用于记录地址，并在用户前显示在Safari中打开(该页面用了ua来判断是安卓还是ios设备，相应的给出是在哪个浏览器中打开)。 回到本项目中，采用ping++，点击支付按钮时，执行如下代码进行跳转：

pingpp.createPayment(json, function(result, err) {  
     console.log(result);
     console.log(err);
});

关键是要找到跳转到支付宝的链接，然后不得不到pingpp.js中看下源码。

pingpp.js中判断是否有_AP对象,没有_AP对象就直接提交表单跳转到支付页面，没有就执行_AP.pay的方法。
是否创建_AP对象,这里需要动态加载ap.js，如果不判断浏览器，在非微信浏览器下(如Safari)支付时都和微信一样显示(请在xxx浏览器打开)，就不是太好。于是要加入如下代码(用于判断是否是微信浏览器)：

  var ua = navigator.userAgent.toLowerCase();  
  if(ua.match(/MicroMessenger/i)=="micromessenger") {  
    var head= document.getElementsByTagName('head')[0]; 
    var script= document.createElement('script'); 
    script.type= 'text/javascript'; 
    script.src= '__ROOT__/Apps/Wap/View/default/enroll/users/pay/src/ap.js'; 
    head.appendChild(script); 
  }

而_AP.pay方法要接收支付宝的url，接收到后，就执行ap.js里的方法跳转到pay.html(因为用了thinkphp框架，这里的wxPay实际上是控制器，如果按照demo直接输入pay.htm?goto='就跳转不到真正的pay.html页面)。所以要ap.js 的location.href="处填上项目所要跳转的地址。
