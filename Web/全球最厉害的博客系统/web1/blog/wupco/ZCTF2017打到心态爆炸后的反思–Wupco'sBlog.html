ZCTF2017打到心态爆炸后的反思

     这个寒假很松懈，没有多练习，开学的伊始我就产生了不好的预感。首先是我要弄个一键备份的功能需要写一个python脚本，各种卡壳，然后一怒之下写php，结果居然和python混淆了....混淆了.....混淆了.......

     然后去华为为老师做点事，在内网看到一篇写的很好的安全文章，瞬间发现自己少了很多灵性。我觉得做安全，尤其是Web方向最忌讳的就是偷懒，安全就是在不断变革，不能跟上时代就会被淘汰。

     这次ZCTF第一道Web100，拿到手各路信息收集了无果后先FUZZ了一下目录（包括swp,swo,swn等文件泄露），然后还是没什么收获，这时师傅已经FUZZ到了1.php是phpinfo页面（我居然没FUZZ到......是时候更新一下脚本了），大致看了一下phpinfo，看到了openssl，看到版本不是心脏出血，又看到是php7+opcache，看了一下http://www.52bug.cn/post-1129.html

需要有个任意文件上传，想到是100分题不可能这么复杂，瞬间又没思路了。这时师傅又FUZZ到了swp文件泄露.......

我当时明明试过了啊喂(#`O′)，有点怀疑是官方刚开始忘了给.......于是拿到了源码。

$flag = $_GET['flag'];
if ($flag != '15562') {
	if (strstr($flag, 'zctf')) {
		if (substr(md5($flag),8,16) == substr(md5('15562'),8,16)) {
			die('ZCTF{#########}');
		}
	}
}

  发现是md5碰撞，1556加密后的第8位开始又是0e，所以找一个同样第八位开始是0e的利用php弱类型判断就好，刚好有个碰撞md5某些位数的脚本...改了一下进行碰撞.....跑了半天没结果......发现是位数搞错了.....我忘记了php的8，16代表的是从第八位到二十四位，而不是第八位到第十六位......这时候俊杰师傅已经碰撞到了.......


       第二题应该是个xss，只有contact.php是动态的，而且在师傅的提醒下，发现上了CSP，这个倒也不难，准备撸payload时候发现拦截了非法字符等等(刚开始以为是textarea XSS)，试了很多，大概猜到黑名单了，这时候我灵机一动，查看了一下.message.php.swp，居然真的有备份文件！！！

 

<?php
