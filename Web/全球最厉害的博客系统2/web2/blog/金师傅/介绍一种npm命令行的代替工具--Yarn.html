介绍一种npm命令行的代替工具 -- Yarn
  • Kylin King
  • 二月 20 2017
  • 前端 服务器

相信使用过NodeJS的都知道npm这个工具，一个Github上的项目，Down下来，直接运行命令行npm install，就安装完第三方依赖模块和包了。但自从npm第三方包node-serialize出现反序列化漏洞，被媒体夸大成NodeJS的漏洞后（传送门: 怎样看待NodeJS模块node-serialize存在的反序列化漏洞?），npm第三方模块的安全性不能保证，因为任何一个noder都可以上传自己的模块。今天看到了Yarn，感觉有代替npm命令行的意思。

什么是Yarn

Yarn官网声称快速、可靠、安全的依赖管理。，从官网来看，应该有以下特点：

  • 极速(可以缓存包，下载过的包就不用从网络中再拉取了，要是项目做的足够多，可以实现离线安装模块/包)
  • 超级安全(这点应该是它的特色，对第三方模块有一个安全检验，保证了第三方模块的质量)
  • 超级可靠(可以保证运行在不同平台MacOS/Windows/Linux下，基本没有什么差别)

安装Yarn

一、MacOS

1.通过Homebrew或MacPorts安装
// 或者使用Homebrew
brew update  
brew install yarn

// 或者使用MacPorts
sudo port install yarn
2.配置

将export PATH="$PATH:yarn global bin"这行添加到配置文件中(可能是.profile或.bashrc或zshrc) 安装完后，最好将镜像源指向国内，运行yarn config set registry https://registry.npm.taobao.org

3.检验是否安装成功

yarn --version，会出来版本号(如0.20.3)

二、Windows

直接下载文件安装latest.msi

三、Linux

对着版本看官网给的方法Linux安装方法
