HTTP代理按匿名功能分类

火日攻天

    代理分类

    1. 透明代理：不具有匿名功能，可以直接查到原ip。
    2. 匿名代理：使用此种代理时，虽然被访问的网站不能知道你的IP地址，但仍然可以知道你在使用代理。
    3. 高匿代理：使用此种代理时，被访问的网站不知道你的IP地址，也不知道你在使用代理进行访问。此种代理的隐藏IP地址的功能最强。

    原理

    根据http包头段，可以看出是否使用代理以及原ip HTTP_X_REAL_IP
    HTTP_X_FORWARDED_HOST
    HTTP_X_FORWARDED_SERVER
    HTTP_X_FORWARDED_FOR
    HTTP_X_REMOTE_PORT

    步骤

    对Squid来说默认配置就是透明代理，不需要配置。如果希望变为匿名代理，修改/etc/squid/squid.conf

    加入forwarded_for off

    这样 X-Forwarded-For这项就会变为unknow，也就是说还是可以看出用了代理，只是不知道是谁用的代理

    如果希望变为高匿代理，加入

    request_header_access Via deny all  
request_header_access X-Forwarded-For deny all

    如果squid是2.x这类的老版本，则加入

    header_access Via deny all  
header_access X-Forwarded-For deny all

    这样就不会有X-Forwarded-For字段，网站就无法判断是否有代理的存在

    对开发者而言

    HTTP字段可伪造，永远不要相信用户的请求，永远不想相信这个输入是可信的，不要用X-Forwarded-For等头部去作为用户的真实ip，获取用户真实的ip在大部分情况下没有意义。
