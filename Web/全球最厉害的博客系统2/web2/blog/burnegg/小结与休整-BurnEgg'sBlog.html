小结与休整

发表于 2016-07-08   |   分类于 记忆   |   暂无评论

小小的总结

  • 从一个漏洞到一堆漏洞：

    1. 对漏洞的发散整理　　==>　　得到相似的漏洞

    2. 对漏洞的深度利用　　==>　　得到更多的信息　　==> 对信息的处理与挖掘

    3. 回到 1.

    4. ....

  • 多个目标的处理:

    1. 从对单个目标的处理中一般化

    2. 用程序表达一般化的思路

    3. 对程序的优化

    4. 最终数据的处理，优化，可视化

    5. 把程序不能正确处理的一些东西呈现出来

  • etterfilter的一些坑:

    1. ip用单引号包裹,其他都只能用双引号包裹

    2. inject frame 的时候会自动处理好tcp的效验,在处理http response的时候replace导致response的长度增加而respond请求头中content-length没有随之改变,导致末尾处的html代码无法进入浏览器渲染

    3. data.data中包含http请求头

  • asp.net和sql:
    order by [字段] asc

字段可控时用]闭合再stack注入
多考虑sql语境并阅读源码
